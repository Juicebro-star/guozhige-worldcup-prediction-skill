# 数据源与查询工作流

> 核心理念(借鉴 nutmeg / machina-sports/sports-skills):**判断具体比赛前先查真实数据,不靠模型训练记忆猜。**
> 本 skill 提供的是"果汁哥的判断框架与表达",赛前事实(伤停、首发、赛果、天气)必须实时获取。

---

## 1. 标准查询流程

1. **确定要查什么**(见第 3 节"维度 → 查什么"对照表);
2. **检索**:优先用联网搜索/可用的数据 API,按"队名 + 关键词 + 时间"组合查询;
3. **交叉验证**:重要结论(如核心球员伤停、是否停赛)尽量两个来源对齐;
4. **代入判断**:把事实接到 SKILL.md 第 5 节分析维度;
5. **标注不确定**:查不到或来源冲突的,写"需临场确认",不要硬编一个数字。

---

## 2. 公开数据源清单

> 说明:以下均为第三方公开来源,本 skill 不拥有、不分发任何数据;使用时自行遵守各来源的服务条款。能力取决于当前对话可用的检索/抓取工具。

| 来源 | 覆盖 | 适合查 | 接入方式 |
|---|---|---|---|
| **Sporttery 官方计算器 / 官方接口** | 竞彩足球/竞彩篮球开售赛事 | 胜平负、让球胜平负、比分、总进球数、半全场、混合过关回报倍数、单关/过关状态、理论奖金计算 | 官方页面 / 可用接口 / 页面脚本解析 |
| **中国足彩网 / 竞彩数据页** | 竞彩足球开售赛事 | 胜平负、让球胜平负、比分、总进球、半全场等实时数字备用源 | 浏览器抓取优先;裸 curl 可能被 WAF 拦截 |
| **ESPN**(公开 web 接口) | 多项运动、足球多联赛/国家队 | 赛程、比分、首发、伤停名单、实时数据、胜率 | 联网搜索 / 公开端点 |
| **Understat** | 五大联赛等 | xG / xGA、射门质量、近期数据走势 | 联网搜索 / 公开 web |
| **FBref** | 极广,含国家队 | 详细统计、xG(数据来自 StatsBomb)、对阵历史 | 联网搜索 / 公开 web |
| **Transfermarkt** | 全球 | 阵容身价、伤停与停赛、转会/出场、市场强弱参考 | 联网搜索 / 公开 web |
| **FPL**(英超) | 英超 | 球员状态、伤停标记、出场预期 | 半官方公开 API |
| **ClubElo** | 俱乐部 Elo | 强弱底座、Elo 走势(实力对比参考) | 公开 API |
| **FIFA / 世界排名** | 国家队 | 国家队强弱底座、排名 | 官方/联网搜索 |
| **官方赛事页 / 权威媒体** | 当届赛事 | 最终赛程、停赛公告、首发确认、天气/赛地 | 联网搜索 |

> 国家队/世界杯场景:**Transfermarkt(伤停身价)+ FBref(统计)+ ClubElo/FIFA排名(底座)+ ESPN/官方页(赛程首发)+ 天气服务(赛地气温)** 组合最实用。

> 方案研究场景:**Sporttery 官方计算器/官方接口优先用于回报倍数和玩法开售状态**。若官方数据抓取失败,可用公开市场赔率临时代替,但必须在输出中写明"本次非官方回报倍数,只做模型示范"。

## 2.5 实时回报抓取优先级

方案研究必须尽力拿到最新真实数字,优先级如下:

1. **Sporttery 官方计算器/官方接口**:优先源。移动端页面路径如 `m.sporttery.cn/mjc/jsq/zqspf/`、`zqzjq/`、`zqbf/`、`zqbqc/`、`zqhhgg/`。
2. **浏览器渲染抓取官方页**:如果接口 TLS、跨域或风控失败,用浏览器打开页面,读取渲染后的 DOM/文本,不要只靠裸 `curl`。
3. **中国足彩网等竞彩数据页**:例如 `cp.zgzcw.com/lottery/jchtplayvsForJsp.action?lotteryId=47&type=jcmini`。该站可能对裸 `curl` 返回 418/WAF,但浏览器抓取可读时可以作为备用真实数字来源。
4. **第三方公开赔率站**:只有当前三类源都失败时使用,并在输出中明确标注"非官方回报,只做模型示范"。

抓取失败处理:

- 不要写"已拿到官方实时数字"。
- 输出中写明失败原因,如"Sporttery 接口 TLS 握手失败""足彩网裸 curl 被 WAF 拦截,改用浏览器读取"。
- 如果多个来源数字冲突,优先官方;无官方时取时间最新且可追溯的来源,并标注来源时间。

## 3. 分析维度 → 要查什么(对照表)

对应 SKILL.md 第 5 节:

| 维度 | 查什么 | 优先来源 |
|---|---|---|
| 实力底座(5.1) | FIFA排名、Elo、近期数据机构强弱 | ClubElo、FIFA、FBref |
| 小组赛状态(5.2) | 三场赛果、净胜球、是否末轮轮休、积分/晋级方式 | ESPN、官方赛事页 |
| 球员状态/轮休(5.3) | 主力是否末轮轮休、替补表现、球星近况 | ESPN、媒体、FBref |
| 伤停/红黄牌/停赛(5.4) | 伤病名单、累计黄牌、红牌停赛 | Transfermarkt、FPL(英超)、官方公告 |
| 战术/结构(5.5) | 阵型、xG/xGA、定位球占比、高压强度 | Understat、FBref |
| 气候/赛地/旅行(5.6) | 赛地气温/湿度/海拔、室内外、旅行距离、恢复天数 | 天气服务、官方赛程 |
| 淘汰赛心理(5.7) | 对阵历史、强队/弱队风格、是否擅长拖加时 | FBref、媒体复盘 |
| 方案研究 | 各玩法回报倍数、单关/过关状态、赔率更新时间 | Sporttery 官方计算器/官方接口;浏览器抓取官方页;中国足彩网等竞彩数据页;最后才用公开市场赔率替代并标注 |

## 4. 数据落地为判断的规则

- **底座 ≠ 结论**:Elo/排名只回答"谁更强",不等于"90分钟稳赢";淘汰赛还要看对手能否把节奏拖慢。
- **伤停看的是"少了什么功能"**,不是"少了几个人":少出球点、少爆点、少防空,比单纯人数更重要。
- **xG 看趋势不看单场**:用近期 xG/xGA 判断进攻效率和防守稳定性,别被一场大胜/惨败带偏。
- **天气进判断**:越热/越湿/越高原 → 越压制长时间高压逼抢 → 热门队更倾向控风险、小胜,而非大胜。
- **查不到就降级**:关键信息缺失时,把结论强度从"强方向"降到"中等/防平",并写明"需临场确认"。

---

## 5. 注意

- 数据时效优先于一切:**临场首发/伤停以赛前 1 小时官方为准**,本 skill 内任何样例数据(见 `examples.md`)都不可直接当作当前赛果。
- 若当前对话无联网/数据工具可用,需明确告知用户"以下基于现有信息,未联网核实",再给判断,避免让用户误以为是最新核实结果。
